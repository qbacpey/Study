Title: Asynchronous I/O structure

• After I/O starts, control returns to user program without waiting for I/O completion.

异步 IO 操作会在执行系统调用之后立刻返回

– System call request to the operating system to allow user to wait for I/O completion.

异步 IO 结构得以实现的前提是有一个设备状态表

– Device-status table contains entry for each I/O device indicating its type, address, and state.

设备状态表中的表项储存了每个 IO 设备当前所处状态、地址以及类型信息

– When interrupt, operating system indexes into I/O device table to determine device status and to modify table entry to include interrupt.

每当硬件中断发生时，OS 都会在 IO 设备状态表中进行寻址，以确定设备状态并修改状态表表项以包含中断
