If the processor takes an interrupt in kernel mode with interrupts enabled, it is safe to use the current stack pointer rather than resetting it to the base of the interrupt stack.

当内核在执行中断处理程序且启用中断时，此时新到来的中断可以直接使用原本中断处理程序的栈

This approach can recursively push a series of handlers’ states onto the stack; then, as each one completes, its state is popped from the stack, and the earlier handler is resumed where it left off

新中断处理程序能使用旧中断处理程序的栈意味着内核中断栈可能会形成递归式的状态结构，每当一个中断处理程序执行完毕，接续的中断状态就会被弹出并恢复其执行状态
