To simplify the kernel design, the hardware provides a privileged instruction to temporarily defer delivery of an interrupt until it is safe to do so.

硬件为了简化内核的设计，提供了推迟中断处理的指令

推迟中断执行的指令是特权指令

On the x86 and several other processors, this instruction is called disable interrupts. However, this is a misnomer: the interrupt is only deferred (masked), and not ignored. Once a corresponding enable interrupts instruction is executed, any pending interrupts are delivered to the processor.

在x86以及其他的一些实现中，暂缓中断执行的指令被称为禁用中断（disable interrupt）指令

执行禁用中断指令之后，往后到来的中断实际上并没有被禁用，仅仅是被推迟了。执行了启用中断（enable interrupt）指令之后，往后的中断才会被分发给处理器

The instructions to mask and unmask interrupts must be privileged; otherwise, user code could inadvertently or maliciously disable the hardware timer, allowing the machine to freeze.
