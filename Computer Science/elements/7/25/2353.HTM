Title: GDB : disassembly

到这里使用的指令实际上就相对注重机器码和源码之间的映射关系了，

考虑到通常情况下一条源码经常对应着多条源代码，因此可以使用 info line location 直接查看这两者之间的映射关系（默认参数是 $pc）：

 (gdb) info line m4_changequote
     Line 895 of "builtin.c" starts at pc 0x634c <m4_changequote> and \
     ends at 0x6350 <m4_changequote+4>.
 (gdb) info line *0x63ff
     Line 926 of "builtin.c" starts at pc 0x63e4 <m4_changequote+152> and \
     ends at 0x6404 <m4_changequote+184>

.

值得注意的是，info line 也能将位置作为参数

同样的 disassembly 也能将 location 作为参数，不同之处在于此命令通常用于打印复数条机器码，根据参数的不同可以分为两种使用情况：

  • 单参数：如果仅向 disas 传递单个参数，这就意味着命令其打印该位置所处的函数栈帧对应函数包含的所有机器码：

     (gdb) disas /m main
     Dump of assembler code for function main:
     5 {
     0x08048330 <+0>: push %ebp
     0x08048331 <+1>: mov %esp,%ebp
     0x08048333 <+3>: sub $0x8,%esp
     0x08048336 <+6>: and $0xfffffff0,%esp
     0x08048339 <+9>: sub $0x10,%esp
     ​
     6 printf ("Hello.\n");
     => 0x0804833c <+12>: movl $0x8048440,(%esp)
     0x08048343 <+19>: call 0x8048284 <puts@plt>
     ​
     7 return 0;
     8 }
     0x08048348 <+24>: mov $0x0,%eax
     0x0804834d <+29>: leave
     0x0804834e <+30>: ret
     ​
     End of assembler dump

    .

    无参数调用 disas 时会将 %pc 作为参数

  • 双参数：在这种情况下 disas 会打印被参数划定的范围中对应的机器码：

     (gdb) disas 0x32c4, 0x32e4
     Dump of assembler code from 0x32c4 to 0x32e4:
     0x32c4 <main+204>: addil 0,dp
     0x32c8 <main+208>: ldw 0x22c(sr0,r1),r26
     0x32cc <main+212>: ldil 0x3000,r31
     0x32d0 <main+216>: ble 0x3f8(sr4,r31)
     0x32d4 <main+220>: ldo 0(r31),rp
     0x32d8 <main+224>: addil -0x800,dp
     0x32dc <main+228>: ldo 0x588(r1),r26
     0x32e0 <main+232>: ldil 0x3000,r31
     End of assembler dump.

    下面涉及到的地址指的是指令的地址，形式包括0x32c4,&main+10,$pc - 8

    • diaas start_addr, end_addr：打印指定机器码区域中的机器码；

    • diaas start_addr, +length：同上，只不过是使用 length 指示偏移量

    ；

最后，如果想要让 disas 呈现源码和机器码之间的对应关系，记得使用 \s 作为参数
