Title: Kernel Threads and User Threads

A kernel thread is the schedulable entity, which means that the system scheduler handles kernel threads.

内核线程是一个可调度的实体，由系统调度器来掌管内核线程

These threads, known by the system scheduler, are strongly implementation-dependent. To facilitate the writing of portable programs, libraries provide user threads.

A kernel thread is a kernel entity, like processes and interrupt handlers; it is the entity handled by the system scheduler.

内核线程也是一个内核实体，就像进程和中断处理器一样

A kernel thread runs within a process, but can be referenced by any other thread in the system.

内核线程在进程内部运行，但是也能被系统中的其他线程引用

The programmer has no direct control over these threads, unless you are writing kernel extensions or device drivers.

程序员不能直接控制内核线程

For more information about kernel programming, see Kernel Extensions and Device Support Programming Concepts.

A user thread is an entity used by programmers to handle multiple flows of controls within a program.

程序员会使用用户线程来控制程序的复数性执行流

The API for handling user threads is provided by the threads library. A user thread only exists within a process; a user thread in process A cannot reference a user thread in process B.

用户线程仅存在于进程之中

The library uses a proprietary interface to handle kernel threads for executing user threads. The user threads API, unlike the kernel threads interface, is part of a POSIX-standards compliant portable-programming model. Thus, a multithreaded program developed on an AIX® system can easily be ported to other systems.

 

On other systems, user threads are simply called threads, and lightweight process refers to kernel threads.
