Title: 继续执行程序

  • 单步推进，这两者唯一的区别仅在于遇见函数时是否进入函数体。这两者也可以接受一个常数参数，表示直接跳转 n 步：

    • step：遇见函数时，跳转到函数中继续步进；

    • next：遇见函数时，不会跳转到函数中继续步进，而是将函数调用当成一条指令

    ；

  • continue：以断点为单位的跳转，如果未指定参数的话会直接跳转到下一个断点的位置；如果指定常数参数的话，会忽略接下来遇到的 n 个断点；

  • finish：跳转直到当前函数调用栈帧被弹出（不过，如果执行的中途会经过断点，GDB 将会被阻塞）；

  • until location：默认情况是跳转直到当前指令对应机器码行号比当前行号高为止，但是还有着各种各样特别的性质：

    • 跨越循环继续执行-1；

    • 当前栈帧被弹出会停止执行；

    • 跨越多层递归实例，因此 until 仅能设置位于本层栈帧（递归实例）中的位置；

  • advance location：类似于 until，但是不会被限制在本层栈帧中，因此不能用来跳过多层递归实例，两者的不同具体看以下代码：

     94 int factorial (int value)
     95 {
     96 if (value > 1) {
     97 value *= factorial (value - 1);
     98 }
     99 return (value);
     100 }

    如果在 96 行执行 until 99 的话，97 行中的所有派生出的递归实例都会被跳过，直到返回本层递归实例中的第 97 行；反之如果执行 advance 99 的话，在最上层递归实例返回时函数就会被停止，而且沿路的所有递归实例经过第 97 行都会被停止。总的来说，until 的目的地是本层栈帧中的位置，advance 则不会受到栈帧的影响
