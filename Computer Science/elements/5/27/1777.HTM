Flag "l" "g" "u" "!" " " 分别表示什么意思？

The symbol is a local (l), global (g), unique global (u), neither global nor local (a space) or both global and local (!). 

A symbol can be neither local or global for a variety of reasons, e.g., because it is used for debugging, but it is probably an indication of a bug if it is ever both local and global. Unique global symbols are a GNU extension to the standard set of ELF symbol bindings.  For such a symbol the dynamic linker will make sure that in the entire process there is just one symbol with this name and type in use
