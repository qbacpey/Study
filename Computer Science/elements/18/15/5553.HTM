Decreasing the number of virtual processors. When the kernel preempts a virtual processor (e.g., to give the processor to a different process), the kernel makes an upcall on one of the other processors assigned to the parallel program. The thread system can then move the preempted user-level thread onto the ready list, so that a different processor can run it.
当内核抢先一个虚拟处理器时（类似于之前提到的 time-slice）内核就需要激活其他给予该程序的虚拟处理器，该程序的线程系统就能将被抢先的用户级线程移动到等待队列中（实际上就是让线程系统的调度器或者说别的线程将这个线程的状态保存到其对应的 TCB 中）这样其他虚拟处理器就能运行这个线程了
