The name of an array usually evaluates to the address of the first element of the array, so array and &array have the same value

通常情况下，数组的名称会被编译器直接视为一个同名的，指向数组首元素的指针，因此 array（首元素地址）和 &array（数组地址）会有相同的值

(but different types, so array+1 and &array+1 will not be equal if the array is more than 1 element long)

但是 array 和 &array 实际上是两个类型不同的表达式，考虑到地址的类型信息由两部分组成（“地址”和“地址元素的大小”）因此尽管这两者都是地址，但是前者地址元素是数组的元素类型，后者的地址元素是数组本身的类型（即数组长度）

另外这里提到的不同类型实际上指的是，本来直接使用数组的名字应该能获得数组类型的值，但是方便起见，他会被直接当成一个指向首元素的指针
