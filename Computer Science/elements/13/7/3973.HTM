Most operating system kernels go one step farther and allocate a kernel interrupt stack for every user-level process (and as we discuss in Chapter 4, every thread that executes user code).

每个线程都有自己的中断栈，此外，线程也被称为用户级进程

When a user-level process is running, the hardware interrupt stack points to that process’s kernel stack.

当线程正在运行的时候，硬件中断栈寄存器指向该线程的内核栈（内核栈 & 内核中断栈？）

Note that when a process is running at user level, it is not running in the kernel so its kernel stack is empty.

大多数操作系统都会为每个用户级进程（线程）分配一个内核中断栈

当用户进程正在执行的时候，硬件中断栈（每个处理器的中断栈？）会指向该进程的内核栈

当进程在用户态运行时，由于它并不在内核中运行，因此内核中断栈是空的
