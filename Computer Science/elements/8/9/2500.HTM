Title: Introduction

下面是一些基本概念：

  • 终端与路由器：终端系统（end system）或者说主机（host）基本上能被认为是我们日常使用的，直接面向普通用户的电子产品。将这些终端系统连接起来的就是由通信链路（communication link）和包交换机（packet switch）所组成的网络，其传输速率（transmit rate）的单位是 bit 每秒；
  • 连接：如果两台计算机能够交换数据，那么就称这两台计算机相互连接（interconnected）；
  • 包：终端系统发送数据时，会将数据分成一个个包（packet），同时分别为这些包加上头字节（header byte）；而在接受端的终端系统中，会将这些包重组成原来的数据；
  • 包交换机：包交换机（packet switch）本质上是一个从接收端接收数据包并向发送端发送出去的设备。典型的包交换机是路由器（router）和链路层交换机（link-layer switch），前者多用于网络核心（network core），后者多用于网络接入（access network）1；
  • 路由和路：路由（route）或者路（path）是网络中由包交换机和通信链路所组成的路径；
  • ISP：全球各地的终端系统之间相互连接的服务，实际上是由网络服务供应商（Internet service provider）来提供的，这些 ISP 自己就是一个由包交换机和通信链路组成的网路2。形式化的说，ISP 就是终端系统接入网络的方式；
  • 实体：实体（entity）表示任何可发送或者接受信息的软件或硬件进程（不一定是主机）；
  • 协议：协议（protocol）定义了网络上任意两个对等（同层）实体之间，所交换的消息的格式及顺序，同时也定义了传输消息时所执行的动作和/或消息的收据（有可能也包括其它事件）3；
  • TCP/IP：网络中的终端系统、包交换机等组件都是通过协议实现对消息的收发控制，其中最重要的两个协议就是 TCP（transmission control protocol）和 IP（Internet protocol），IP 协议控制的是终端系统和路由器之间收发包的格式；
  • 网络标准：网络标准（Internet standard）由 IETF（网络工程任务组）进行指定，其标准文档被称为 RTFs（计算机与通信文件），正是这一份文档定义了诸如 TCP,IP,HTTP 等各种各样的协议；
  • 套接字接口：套接字接口（socket interface）规定了运行在一个终端系统上的应用要如何使用网络组件将数据分发到位于另一个终端系统的应用上，简而言之就是发送消息的应用必须遵守的一些规则；
  • 内网外网：注意，内网（Intranet）和外网（Outhernet）的概念和 LAN 不一样，前者着重强调权限方面的问题；

规模

如果按照网络规模对网络进行划分的话，能分为三种：

  1. LAN（MAN）：局域网，强调某一块较小的地区中终端系统相互连接所依赖的网络；
  2. WAN：广域网，相比于局域网来说，所涵盖的区域范围更大了；
  3. Internet：因特网，囊括整个世界范围；

边界

如果按照网络的边界进行划分的话，网络能分为两种：

  1. 内联网（intranet）：内联网一般是某个企业的私有网，包括许多相互连接的局域网，可能租用部分广域网；
  2. 外联网（extranet）：外联网可以被看做是某个公司暴露给用户的部分网络；

网络边缘

先介绍一些概念：

  1. 主机：考虑到终端系统经常作为网络应用运行的物理载体，因此往往不加区分的混用主机（host）和终端系统（end system）4；
  2. 客户端和服务器：有时主机会被进一步区分为客户端（clients）和服务器（servers），服务器往往坐落于大型的数据中心（data center）；

接入网

接入网（access network）指的是将终端设备和第一个遇到的路由器（也称为边缘路由器，edge router）连接起来的网络5，主流的接入网方式有两种：

  1. 数字订阅线路（digital subscriber line）：依托电话公司的基础设施搭建的网络；

    • 不共享；

  2. 电缆（cable）：依托有线电视公司的基础设施搭建的网络；

    • 共享；

本小节的重点在于理解终端用户接入网络的模式，不必拘泥于接入网的具体形式：终端系统与猫相连，由它将数字信号转化为模拟信号。随后，这股模拟信号会和原有的信号（电话信号或有线电视信号）合流，传输到底层 ISP 的服务站之后，相关的设备会将这股合流信号进行拆分并传输到其各自的目的地

DSL

DSL 网络接入服务一般是由电话公司提供的，因此一般情况下一个用户的电话公司同时也是其 ISP

DSL 一般仅会被用于短距离通信，即 CO 和用户住宅之间：

用户会使用自己的 DSL modem 将数字信号（digital data）转换成专用于传输的高频率双音频（high-frequency tone）以便通过电话线传递给位于中心站（central office）的 DSLAM，被传输过来的模拟信号又会被 DSLAM 重新转换成数字信号

DSL 作为异步接入网络，下游速率会远大于上游速率

由于住宅电话线会同时将数据和传统的电话信号都传递给中心站6，因此这种接入方式会使得单个的 DSL 链接显得被分成三份一样：

  • 在用户侧，一个分离器（splitter）会将经由 DSL 到达的信号分成数据信号和电话信号，也会将这两者通过 DSL modem 合二为一，发送到 CO 中；
  • 在 ISP（电话公司）侧的 CO 中，DSLAM（DSL 接入复用器）会将传输自 DSL 的信号重新分成数据信号和电话信号，并将其传输到相应的处理器中；

Cable

不同于 DSL，有线互联网接入（cable Internet access）的网络通信设施并非由电话公司提供，它会使用有线电视的基础设备来实现网络通信

对应的，有线互联网中的 “DSL modem” 被称为 “Cable modem”；CO 则被称为 Cable head end；DSLAM 则被称为 CTMS（cable modem termination system）；而这整个系统很多时候由于其物理特性，被称为 HFC（hybrid fiber coax）7：

类似于 DSL 的异步特性，Cable modem 会将 HFC 网络分成两个频道：下游和上游，下游频道往往比上游频道的传输速率更高：

  1. 更快的下游（downstream）频道将会被分作下载之用，从有线电视前端向用户传递数据会使用下游频道；
  2. 相对较慢的上游（upstream）频道将会被分作上传之用，从用户发送数据到有向电视前端会使用上游频道；

特性

有线互联网是一个共享广播媒体（shared broadcast medium），一旦有多个用户同时从网络中下载什么东西，每个用户实际上接收到的数据量往往会比有线电视公司宣称的要少（毕竟被其它人分走了）

此外，由于上游频道也是共享的，因此需要一个分布式的多接入协议（distributed multiple access protocol）来协调传输和避免拥塞（coordinate transmissions and avoid collisions）

物理媒介

物理媒介（physical media）被分为两种：导向性媒介（guided media）和非导向性媒介（unguided media），无论是哪种媒介，本质上都是通过电磁波传递数据的：

  • 导向性媒介指的是电磁波会随着固体介质（光纤、双绞线、同轴电缆）进行传递；
  • 非导向性媒介指的是电磁波会随着非固体介质（大气）进行传播（以无限 LAN 或者数字卫星频道）；

值得注意的是，卫星信号也算是物理媒介

双绞铜线（Twisted-Pair Copper Wire）是应用最广泛也是最便宜的导向性媒介，一条双绞线中往往有多对线，每一对线代表着一条通信链路：

非屏蔽双绞线（Unshielded twisted pair，UTP）通常被用于建筑物内部的计算机网络，例如 LAN

同轴电缆

同轴电缆（Coaxial Cable）经常被用于有线电视系统中，也经常被用在导向性共享媒体（shared medium）中。让终端系统直接和该同轴电缆相连就能接受到经由该电缆发出的所有数据

光纤

光纤（optical fiber）是能传输光脉冲的介质，每次脉冲代表一个 bit

光纤的三个优势：

  1. 免疫电磁干扰；
  2. 低信号衰减；
  3. 难以窃听；

网络核心

网络核心（Internet core）能将终端系统连接起来，而网络边缘指代的是各个终端系统（边缘路由器）

本小结的重点在于理解两种网络交换模式：

  • 分组交换（packet switch）：

    • 以数据包为传输单位；
    • 各网络组件按需使用网络资源；
    • 无连接服务，性能不受保证；

  • 电路交换（Circuit Switching）：

    • 使用两种特别的技术实现单链路多连接（FDM、TDM）；
    • 网络会为指定数量的链接预留网络资源；
    • 面向连接的服务，拥有受保证的性能；

在此之前，先定义一下连接（connection）：连接指的是终端设备在发送任何数据之前，先使用协议建立一条端到端的逻辑或者物理连接。面向连接的协议（connection-oriented connection）具有以下服务形式：

  • 握手（handshake）：协调两个待连接主机的参数；
  • 数据传输（data transfer）：TCP 协议；
  • 连接终止（connection termination）；

分组交换

消息（message）实际上指的是源终端系统发送给目标终端系统的数据，消息即可以用于实现控制相关的功能，也可以仅用于携带数据。为了能将消息发送输出，源终端系统需要将消息分成一个个包8

  • 无连接的分组交换（Connectionless packet switching）也被称为数据报交换（datagram switching）：

    • 数据包中含有目的地址、源地址、端口号等信息；
    • 尽管上述信息免除了预先建立链接的需要，但是这意味着每个包需要携带更多数据；
    • 动态决定路由，因此不同数据包尽管起始地址相同，往往也会选择不同的路由；
    • 乱序到达的数据包可能会在目的节点被重新组装；
    • 以太网、IP 协议、UDP 协议都是无连接的服务；

  • 面向连接的分组交换（Connection-oriented packet switching）也被称为虚拟电路网络（virtual circuit network）：

    • 在正式传输任何数据之前都需要建立通信所需要的参数；
    • 仅有在连接建立阶段需要使用无连接服务（地址信息）发现路由；
    • 正式交换数据阶段无需地址信息，取而代之的是链接声明符（因此数据包会更小）；
    • X.25、TCP、ATM 都是面向连接的服务；

排队延迟、丢包

鉴于每个包交换机都有着数个链路与之链接，包交换机为每个链路都配备了一个输出缓存（output buffer）或者说输出队列（output queue），用于暂时储存需经此链路发送出去的数据。如果该链路繁忙，待发送的包必须在对应链路的输出队列中等待一段时间。因此，除了储存和转发延迟，包还会受到排队延迟（queuing delay）的影响9

网络拥塞（congestion）会在路由器处理包的速率不及包被发送的速率时发生10

考虑到实际上输出队列的空间是有限的，如果当某个包到达包交换机时发现其输出队列已经满了，就会发生丢包（packet loss）。丢包无非有两种解决方案：舍弃掉队列中的包或者舍弃掉刚到达的包11

转发表和路由端口

很多时候，一个路由器往往有多个链路与之相连，那么，路由如何决定将包分发到哪一条链路中？

  • 首先，每个终端系统都拥有着一个 IP 地址，类似于建筑的地址一样，IP 地址也有着一定的层级结构；
  • 其次，当某个终端系统想要将某个包发送到某个终端系统时，源终端系统会将目标终端系统的 IP 地址放到包的头字节中12；

基于以上网络特性，路由器会使用转发表（forwarding table）根据目标地址的一部分或者整个目标地址，确定应该将这个包发送到哪一个出站链路（outbound links）中

那么，每个路由器的转发表又是通过什么样的方式设定的呢？简单来说就是通过一系列的路由协议自动设定的

虚拟电路

下面将谈谈虚拟电路

虚拟电路和电路交换异同点

电路交换相比于虚拟电路能提供稳定的比特传输速率和延迟，这是因为虚拟电路会因为以下因素受到影响：

  • 不同节点包队列长度不一；
  • 应用所保证的比特速率不一；

虚拟电路的优势

基于分组交换的虚拟电路在传输数据的过程中，总是会采用相同的路由，经过相同的节点：

  • 路由器不会对数据包执行路由算法，只有一个小的虚拟频道标识符（VCI）会被检查（而不是 IP 地址）；
  • 路由信息仅在链接建立时会被传输，随后路由器就会保存它们的状态；
  • 链路以及路由资源有可能会被预留；
  • 只有在链接建立时每个节点才会对数据包执行路由；

虚拟电路的实现

虚拟电路包括：

  • 从源节点到目标节点的路径；
  • VC 号，路径上每一条链路都有着自己的 VC 号；
  • 沿路的转发表中会有相关的表项储存 VC 号；

整个转发过程的逻辑是：路径上的数据包每到达一条新的链路，其 VC 号都会被替换成对应链路的 VC 号

电路交换

电路交换（Circuit Switching）中会话所需要的资源会被预留，也就是说，尽管网络中有剩余的资源，但是不能像分组交换一样，一份会话将这所有的资源予以独占13。此外，就算某条链接中并没有发送或接收任何数据，这条链接也会被一直保留直到显式的断开连接14

为了实现这一特性，电路交换网络中每一份会话都需要建立一条线路（circuit）：

  • 线路需要预先建立，这也就意味着两个主机之间形成了端到端（end-to-end）的链接；
  • 线路的建立意味着路径（path）上的交换机和各种设备都会为此线路保持链接状态；
  • 网络或为线路预留部分固定的传输速率，也就是说，传输方和接收方至少被保证能以一定的速率传输数据；

那么，一条链路（link）是如何容纳多条线路（circuit）的呢，这就涉及到以下两个技术：

  • 频分复用（frequency-division multiplexing，FDM）；
  • 时分复用（time-division multiplexing，TDM）；

电路交换与分组交换

实践中，具体在这两种交换方式中选择哪一种很大程度上取决于网络源主机传输数据的模式，如果源节点会以恒定比特率（CBR）传输数据，选择电路交换可能更好，这是因为如果源主机的比特率恒定的话

此外，面向连接式传输实际上和电路交换不是相互绑定的，分组交换也可以实现面向连接式传输。因此电路交换和分组交换两者在延迟上最大的差别是从源主机到目标主机的路由上，所有链路对应的路由器是否为存储转发式传输。如果是分组交换（无论是数据报网络还是虚拟电路网络），那么路由上每一个路由器的传输延迟都要被计算在其中，电路交换则不用，只需要将链接建立的时间算在里边即可

网络的网络

既然 ISP 能将不同的终端用户连接起来，那么这些 ISP 之间又是如何连接起来的呢？

答案是通过建立网络的网络（network of network），为了更好解释什么是网络的网络，我们不妨以逐渐增加复杂度的形式，展现今天互联网的样子

网络结构一

最简单，直接的方法是将所有直接与终端用户直接对接的 ISP 用一个更大的、全球性的 ISP 连接起来

由于底层 ISP 实际上需要向全球 ISP 支付向其它底层 ISP 通信时所产生的费用（毕竟是这个全球运营商将这两个 ISP 连接起来的）因此我们不妨将这个拥有交换数据需求的 ISP 称为消费者（customer），将连接各底层 ISP 的全球 ISP 称为提供者（provider）

网络结构二

很显然，当网络的规模继续增大，单个的全球 ISP 已经不足以满足市场的需求了。为此自然会出现多家全球性的 ISP，这些 ISP 彼此相连，构成更加复杂的网路

网络结构三

有时候，在一些网民数量巨大的地区，仅一个全球 ISP 也不足以消耗所有的接入网络需求。基于此，出现了地区性 ISP（regional ISP）用以将底层 ISP 连接起来，再将数据转发到第一层网络（tier-1 ISP）中15

总的来说，网络结构三大致呈：终端系统 => 底层 ISP => 地区性 ISP => 第一层网络 ISP

网络结构四

当然，今天的网络很显然不仅仅是简单的、由 ISP 运行商形成的层级结构这么简单。除了层级结构本身，网络中还包括各种各样的物理基础设施：入网点（points of presence）、多宿主（mult-homing）、对等连接（peering）以及互联网交换中心（internet exchange points）

入网点

PoP 实际上就是一个提供者（provider）放置路由器的地方，消费者能在这里接入提供者的网络

多宿主

Mult-home 指的是一个终端系统或者消费者选择接入了多个提供者，从而确保其中一个提供者出现问题时不至于没有网络可用

对等连接

对等连接是一种为了减少网络通信花费而使用的链接方式

原本如果两个底层 ISP 想要相互连接的话必须先要将数据上传到地区 ISP 上，再由地区 ISP 将数据转发到另一个底层 ISP 上。但如果这两个底层 ISP 发现彼此之间直接相连并通信的开销要远小于之前采用的方案，那么很自然的这两个地区 ISP 就能直接通信，这就是所谓的对等连接（一般来说，对等连接意味着并不需要相互支付网络连接费用）

互联网交换中心

IXP 实际上就是一个由第三方所提供的建筑物，在这个地方许多 ISP 可以实现相互之间的对等连接

网络结构五

网络结构五在网络结构四的基础上添加了内容分发网络（content-provider network），Google 就是一个这方面的例子。Google 的内容分发网络是一个独立的网络体系，它通过让全球各地的 Google 服务器（这些小型的数据中心通常位于 IXP 中）直接相连，绕过第一层网络，让终端用户能更快的访问到位于其它地区的 Google 内容：

分组交换的延迟、丢包和吞吐量

本小节主要介绍分组交换网络传输数据时会遇到的问题：

  • 总节点延迟：节点处理延迟、排队延迟、传输延迟、传播延迟；
  • 包传输速率和包到达速率将如何影响排队延迟；
  • 端到端传输延迟从总体上看是什么情形；
  • 不同传输链路的吞吐量对系统整体吞吐量的影响；

延迟的种类

如上所见，分组交换网络中延迟主要分成四种：

  • 节点处理延迟（nodal processing delay）；
  • 排队延迟（queuing delay）;
  • 传输延迟（transmission delay）；
  • 传播延迟（propagation delay）；

上述四个延迟的总和就称为总节点延迟（total nodal delay）

处理延迟

处理延迟主要由两部分组成16，对路由器的最大吞吐量有着极大的影响：

  • 检查包的头字节；
  • 确定应当将包转发到哪一条链路中；

高速路由器的处理延迟数量级是微秒

排队延迟

排队延迟实际上就是包到达对应链路的输出缓存（output buffer）之后，等待路由器传输的这段时间。主要为以下两个因素影响17：

  • 输出缓存中当前正在排队的包的数目和长度；
  • 路由器的传输速率；

实际中排队延迟的数量级大概在微秒和毫秒之间

传输延迟

传输延迟指的是将整个包中的所有比特都发送到链路所用的时间

传输延迟的数量级大概在微秒和毫秒之间

传播延迟

传播延迟指的是将整个包中的所有 bit 传输到另一个路由器所需要的时间，取决于链路所用的传输介质，并不取决于路由器的传输速率。仅有当包中最后一个 bit 也到达目的地路由器之后，该路由器才会开始处理这个包

万维网中的传播延迟的数量级大概是微秒

网络分层

为使网络协议以结构化的形式组织，网络设计者会将这些协议（以及实现这些协议的软件和硬件）以协议层（layer）的方式组织起来，并使每个协议都属于特定的一层

如果想要以层级结构的方式看待整个网络的话，我们不妨先给出两个定义：

  • 服务（service）是某一层提供给上一层的功能；
  • 某一层的服务模式（service model）也就是上述定义的简称；

那么，每一层究竟要如何提供特定的服务呢？

  • 在本层中执行特定的功能；
  • 直接使用下一层所提供的服务；

协议层还具有两个特点：

  1. 协议层既能被软件实现，也能被硬件实现；
  2. 各协议层并非全部位于某个主机上或者说包交换机上，而是分散的位于不同主机、包交换机、猫等各种各样的网络物理组件上；

再谈协议

网络上的实体（entity）指的是任意两个能够进行通信的软件或硬件进程。协议（protocol）则定义了网络上任意两个对等实体之间，所收发信息的格式和顺序，以及接受和传输信息时应当执行的动作或者应当发生的事件。

在协议的控制下，两个对等实体间的通信最终能成为提供给上层的服务。更确切的讲，上层只能看见下层提供的服务而不能看见下层所用的协议（服务是垂直的，协议是水平的）

服务访问点（SAP）是上下层交互的地方，上下层通过交换原语进行通信（不涉及到网络通信，由操作系统进行管理），而相邻层之间的实体通信需要借助网络进行

协议必须将所有不利的条件都事先估计到，不能假定一切都是正常的，不能假设一切都是正常和非常理想的，也就是需要仔细检测协议是否能应付各种异常协议

网络协议栈

网络协议栈（protocol stack）作为各协议层形成的层次结构，大致呈现如下形式：

各层的作用基本可归结为：

  • 应用层：将用户产生的数据打包成消息并发送到传输层；

    • 针对一类具体应用提供的协议；
    • FTP，SMTP，HTTP；

  • 传输层：使用 TCP/UCP 协议控制数据传输的速度等各种各样的参数，从而提高网络的可依赖性；

    • 主机到主机，端口到端口；
    • TCP，UDP；

  • 网络层：使用 IP 协议为数据添加一些元信息，并使用路由协议规划路由；

    • 源节点到目标节点；
    • IP，路由协议；

  • 链路层：根据数据包对应链路的类型，确定使用何种协议传输数据包；

    • 相邻节点之间的通信；
    • PPP，Ethernet；

  • 物理层：根据链路的物理媒介，选择相应的传输协议；

本质上，网络发送数据的过程实际上是一个不断将数据封装并加以传输的过程18：

  • 应用层将消息传递给传输层，即所谓应用层消息（application-layer message）；
  • 传输层对消息进行封装并在其上添加额外的信息19，生成传输层段（transport-layer segment）；
  • 网络层会将段加以封装，同时也在其中添加额外的数据20，生成网络层数据报（network-layer datagram）；
  • 链路层也会将额外的数据添加进去并生成链接层帧（link-layer frame）；

总的来说，每层包都由以下两个部分组成：header field 和 payload field（后者就是从上层接受到的数据包）

应用层

应用层（application layer）是网络应用及相关的协议所处的位置，网络应用会借助相关的协议与位于其它终端系统上的网络应用交换数据包

应用层中包含非常多的协议：

  • HTTP：提供网页文档的请求和传送；
  • SMTP：提供电子邮件的传送；
  • FTP：提供两个终端系统的文件传送；

应用层中的数据包被称为消息（message）

传输层

传输层（transport layer）提供不同终端系统间消息传递的服务

传输层中典型的传输协议有两种，TCP 和 UDP：

  • TCP 提供的服务是面向连接型（connection-oriented）的服务，功能主要包括以下四个：

    • 提供可靠，有序的字节流传输；
    • 流控制（flow control）21；
    • 将相对较长的消息分割成相对较短的段（segment）；
    • 提供拥塞控制（congestion-control）结构22；

  • UDP: UDP 提供的服务是无连接型的，相对于 TCP，其功能简单很多。既没有流控制，也没有拥塞控制，因此就不保证任何可依赖性；

如果传输层使用的是 TCP 协议，那么其中的数据包被称为报文段（segment）；如果使用的是 UDP 就称为数据报（datagram）

网络层

网络层（network layer）的功能是将一个主机上的数据报（datagram）传输到另一台主机上（从源节点到目标节点），本层的协议除了包括用于确定数据包路由（自起点到终点的路由）的路由协议，还包括 IP 协议

本层中的 IP 协议定义了数据报中的字段，以及终端系统以及路由器应当怎样处理这些字段

网络层中的数据包被称为数据报（datagram）

链路层

如果网络层想要将数据报转发到路由上的下一个节点，就必须借助链路层（link layer）的帮助：每个网络组件中的网络层都会将数据报传输给其下的链路层，再由它将数据报传送给下一个节点中的链路层。数据报到达后，节点链路层则会将接受到的数据报传送给其上的网络层

但是链路层具体会提供什么服务很大程度上取决于两节点之间所使用的链路形式及对应的协议。对于某一个数据报来说，其可能在一条链路上被以太网对应的链路层协议所处理，在另一条链路上被 PPP 对应的链路层协议所处理。因此，位于不同路由器上的网络层往往会收到不同链路层协议的服务

链路层中的数据包被称为帧（frame）

物理层

物理层（physical layer）传输对象的单位是帧中的单一一个 bit。类似于链路层，本层的协议（及服务）也依赖于链路的具体类型，更进一步的，甚至会依赖于链路上具体的传输介质（双绞线，光纤都有各自对应的协议）；所使用的协议不同，链路上 bit 的传输方式自然也不同

物理层的数据报就简单的被称为包（packet）

封装

网络中的物理组件并不会实现全部的协议层：

  • 主机实现了网络协议栈中的全部层（因特网架构将非常多的复杂度放在了网络边缘中）；
  • 路由器仅实现了协议栈的底下三层（物理层、链路层、网络层）；
  • 链接层交换机仅实现了协议栈的底下两层（物理层、链路层）因此它并不能识别 IP 地址，而只能识别第二层的地址（以太网地址）；
  • 集线器仅实现了最底下的一层（物理层）；
