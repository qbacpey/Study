Title: What's the difference between DMA Controller and I/O processor : Direct Memory Access(DMA):

DMA provides this capability to carry out memory specific operations with minimal CPU intervention.

When any I/O device needs a memory access. It sends a DMA request(in form of interrupt) to CPU. 

当任何 IO 设备需要执行内存访问操作的时候，它会以中断的形式向 CPU 发送一个 DMA 请求

 

 

 

CPU initiates the the transfer by providing appropriate grant signals to the data bus. And passes the control to the DMA controller which controls the rest of the data transfer and transfers the data directly to I/O device.

CPU 通过向数据总线提供合适的许可信号来初始化传输，随后将内存（或者相关硬件）的控制权转给 DMA 控制器

DMA 控制器则负责剩下的数据传输工作，即直接将数据传递给 IO 设备

 

During this time, CPU continues with other instructions. Once the Read/Write operation in completed (or any exception is occurred )the DMA controller initiates an interrupt and notifies the processor about the status of read/write operation.

当 IO 操作完成，DMA 控制器就会向 CPU 发出中断，以告知读写操作的状态

In this way the read/write operation is also carried out and CPU also executes some other instruction during that time. However, initialization of DMA still requires CPU intervention. And so the overall performance is maximized.
