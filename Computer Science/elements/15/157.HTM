First, we provide some background on the x86 architecture. The x86 is segmented, so pointers come in two parts: (i) a segment, a region of memory such as code, data, or stack, and (ii) an offset within that segment.

由于x86的内存结构是分段化（segmented）的，因此其内存指针由两部分组成：段（segment）所在位置和偏移量（offset）

The current user-level instruction is a combination of the code segment (cs register) plus the instruction pointer (eip register).

用户指令的当前位置由 `CS` 寄存器和 `EIP` 寄存器共同指定

`CS` 寄存器所指向的内容是代码段所在位置（Code Segment）

`EIP` 寄存器所指向的内容是指令指针的偏移量

Likewise, the current stack position is the combination of the stack segment (ss) and the stack pointer within the stack segment (esp).

当前栈的所在位置由 `SS` 寄存器和 `ESP` 寄存器共同指定

`SS` 寄存器所指向的内容是当前栈的所在位置（即可能是用户栈，也可能是内核栈）

`ESP` 寄存器所指向的内容是栈指针的偏移量

The current privilege level is stored as the low-order bits of the cs register rather than in the processor status word (eflags register).

当前所处特权等级会被保存到 `CS` 寄存器的低比特位上

处理器状态字（Processor Status Word）会被保存到 `EFLAG` 寄存器中

The eflags register has condition codes that are modified as a by-product of executing instructions; the eflags register also has other flags that control the processor’s behavior, such as whether interrupts are masked or not.

`EFLAG` 寄存器中主要保存的是一些控制比特位，比如说当前中断是否被禁用
