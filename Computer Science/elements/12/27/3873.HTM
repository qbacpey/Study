Thus, kernels typically allocate a very small fixed sized region for each thread stack, e.g., 8 KB by default in Linux on an Intel x86. The kernel stays within this bound due to an important kernel coding convention: buffers and data structures are always allocated on the heap and never as procedure local variables

由于内核线程地址空间是物理地址空间，因此内核线程栈往往非常小（x86 是 8 KB）这主要是由于内核线程的缓冲和数据结构永远会被保存到堆中（因此就不需要这么大的栈）
