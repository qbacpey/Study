We want the user to run a specific application program without access to the shell. We could change the sign-in program in /etc/passwd, but maybe we want environment setting to be used from start-up files. So, in (say) .profile, the last statement says something like: 

 exec appln-program

so now there is no shell to go back to. Even if appln-program crashes, the end-user cannot get to a shell, because it is not there - the exec replaced it

如果我们想让用户在不访问 shell 的情况下启动某个应用程序，可以怎么做（假设该程序名为 appln-program）? exec appln-program
