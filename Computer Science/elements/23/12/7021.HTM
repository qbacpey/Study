Title: G.3 Function and Parameter Attributes

These macros deﬁned in <debug.h> tell the compiler special attributes of a function or function parameter. Their expansions are GCC-speciﬁc.

加在函数原型或者函数参数之后的宏定义在 debug.h 文件中

UNUSED Appended to a function parameter to tell the compiler that the parameter might not be used within the function. It suppresses the warning that would otherwise appear.

如果在函数形式参数后面加上一个 UNUSED 宏，那么就会告诉编译器这个形式参数可能不会在函数中被使用，它的作用是压制编译器的警告

NO_RETURN Appended to a function prototype to tell the compiler that the function never returns.

It allows the compiler to ﬁne-tune its warnings and its code generation.

如果在函数原型后面加上一个 NO RETURN 就相当于告诉编译器这个函数永不返回。它的作用是允许编译器能更细粒度地呈现其警告以及执行代码生成

NO_INLINE Appended to a function prototype to tell the compiler to never emit the function in-line.

Occasionally useful to improve the quality of backtraces (see below).

如果在函数原型后面加上一个 NO INLINE，就相当于告诉编译器不要内联此函数。这个宏有时候会被用作提升 backtraces 的质量

PRINTF FORMAT (format, ﬁrst) Appended to a function prototype to tell the compiler that the function takes a printf-like format string as the argument numbered format (starting from 1) and that the corresponding value arguments start at the argument numbered first. This lets the compiler tell you if you pass the wrong argument types.

告诉编译器这个函数会接收一个类似于 printf 的参数作为其参数
