On the x86, for example, interrupt vector table entries 0 - 31 are for different types of processor exceptions (such as divide-by-zero); entries 32 - 255 are for different types of interrupts (timer, keyboard, and so forth); and, by convention, entry 64 points to the system call trap handler

中断向量表被称为（interrupt vector table）中所指的中断处理程序仅包括硬件中断吗？不是的，实际上三种中断对应的处理程序都在其中

中断向量表中有几个表项是为系统调用处理程序预留的？一个，下标为 64

尽管中断向量表被称为（interrupt vector table）但是其中的表项所指向的处理程序却包括中断处理程序、处理器异常处理程序、系统调用陷阱处理程序
