Title: 2.8 Implementing Upcalls : Asynchronous I/O notification.

Most system calls wait until the requested operation completes and then return.

大多数执行系统调用的程序会一直等待，直到系统调用执行完毕

What if the process has other work to do in the meantime? One approach is asynchronous I/O: a system call starts the request and returns immediately.

通过系统调用执行的异步 IO 会在请求发出后立即返回

Later, the application can poll the kernel for I/O completion, or a separate notification can be sent via an upcall to the application when the I/O completes

异步 IO 中的应用程序既可以轮询内核以确认 IO 是否完成，也可以让内核在 IO 完成时向应用发送一个上行调用
