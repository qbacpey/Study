Title: Introduction : 操作系统发展

早期的操作系统基本都是单任务的，没有什么介绍的必要

批处理操作系统（batch operating system）指的是当某项任务正在进行的时候，操作系统会使用一个名为直接内存访问（direct memory access，DMA）的进程，驱动 I/O 设备将之前/之后执行的任务传输出去/进来。当 I/O 设备完成传输任务之后，硬件会中断当前进程的处理，使操作系统能启动另一条 DMA 进程

批处理操作系统很快被拓展出了多任务（multitasking）功能，多个程序将会在同时被读取到内存中，一旦之前的某个任务暂停执行，后续的任务将会立刻使用闲置的处理器操作系统发展-1。由于多任务功能就意味着多程序共享处理器，这背后自然需要良好的进程间独立功能予以支撑，因此在此期间，计算机设计者添加了相关的硬件内存保护以减少进程独立所带来的的额外开销（Overhead）

此外，批处理操作系统还需要面对一个棘手的问题，如何调试自身？答案是使用虚拟机。抽象虚拟机（abstract virtual machine）可以让操作系统像其他应用一般在托管操作系统（host operating system）上运行

分时操作系统（time-sharing operating system）支持以交互而不是批处理的方式和系统进行通信，每次操作交互设备（点击鼠标、敲击键盘）都会中断处理程序，使其从交互设备中读取信号并将其排到操作系统的队列中
