Figure 2.9 summarizes the various states of a process’s user and kernel stacks:

  • If the process is running on the processor in user mode, its kernel stack is empty, ready to be used for an interrupt, processor exception, or system call.
    当进程正在用户态执行时，内核栈为空If the process is running on the processor in kernel mode — due to an interrupt, processor exception or system call — its kernel stack is in use, containing the saved registers from the suspended user-level computation as well as the current state of the kernel handler.
    当进程处于内核态时，内核栈中会包含进程被暂停时的所有状态，同时也包含内核处理器的当前状态 If the process is available to run but is waiting for its turn on the processor, its kernel stack contains the registers and state to be restored when the process is resumed.
    当进程等待CPU空闲时，内核栈中会包含用户进程的状态

  • If the process is waiting for an I/O event to complete, its kernel stack contains the suspended computation to be resumed when the I/O finishes.
    当进程等待I/O完成的过程中，内核栈中会包含当I/O完成时需要接续的计算过程（这应该就是之前提到的“上下文”）
