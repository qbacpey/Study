Title: Introduction : 定义

OS（operating system）是一层帮助用户或者应用程序管理硬件资源的一层软件，通用OS的结构大致如下：

  1. Hardware 除了包含处理器、内存和一系列用以保存数据或者和外界交流的设备以外，还含有一些可被用于错误提交和同步的原始硬件，OS 会调度应用程序对于底层硬件的访问；

  2. Kernel-Mode 由两个部分组成：第一个部分依赖于硬件（device-specific）服务，其作用是管理硬件设备。第二个部分不依赖于硬件（即不同的硬件实现都能使用这部分的 OS）这是 OS 面向应用程序的部分；

  3. User-Mode 中各应用程序的运行环境独立，这主要是为了能将恶意或者充满错误的应用程序与正确的应用程序以及 OS 隔离开定义-1；

  4. 各应用程序之所以拥有独立的运行环境，是因为这种抽象架构有助于提高程序的稳定性、性能（消除硬件限制）以及可用性（经由通用服务）；

  5. 此外，OS 的一部分也能作为系统库链接到应用程序中；

那么，OS的角色（或者说其工作）具体是什么呢：

  • Illusionist（幻术师）— OS 对硬件进行抽象以简化对于应用程序设计者的要求：

    • 使编码操作无需关注硬件细节（内存的大小、核心数目）；

    • （经由 OS 的资源调度）能更加灵活的利用硬件资源；

    • 高级对象：文件、用户、消息；

  • Referee（裁判）— OS 将协调各进程对于硬件的使用情况：

    • 协调各应用程序对硬件资源的使用，既包括高效利用也包括使用许可；

    • 防止某应用程序的错误影响到另一应用程序；

  • Glue（胶水）— OS 以提供共用服务（common services）的方式简化应用程序之间的共享操作：

    • 对于不同计算机上所使用的不同 IO 设备，由于这些计算机上针对硬件和应用程序的共用服务都是相同的，因此同一应用程序在这些计算机上都能被使用；

Illusionist

OS 被称为 Illusionist 的原因可从下图管窥一二：

显而易见的是，硬件实现（体现为 ISA）经过层层抽象才为应用程序所用：

  • 将 ISA 抽象为线程、地址空间、文件、套接字：

    • Thread：线程是对处理器的抽象。寄存器，数据通路（DataPath）等都会经由操作系统抽象成线程；

    • Address Spaces：地址空间是对内存的抽象，使得尽管实际上有多个程序共同使用这一片内存，但是在单个程序的认知中这整片内存都为其所用；

    • Files：文件是对硬盘中大小各异的数据块（Block）的抽象；

    • Sockets：套接字是对网络设备的抽象；

  • 将 OS 呈现物抽象成进程illusion-2：

    • 进程可被认为是一层具有提供自操作系统限制权限的虚拟环境（或者说虚拟机）；

    • 应用程序能基于进程运行而无需关注硬件实现的细节；

    • 可以将进程当成是一个能提供诸如线程、地址空间的环境；

经编译的程序就能在进程（抽象）上运行了illusion-1

回到刚才的问题，为什么操作系统会被称为幻术师呢？操作系统通过将硬件抽象成虚拟环境，使应用程序能以一种更加高效的方式利用硬件资源

Virtualization

形式化的说，OS 通过虚拟化（Virtualization）实现对硬件资源的抽象化。它能给所有应用程序一种 “错觉”：内存是无限的、网络带宽是足够的、自己能独享计算机上的一个处理器。不过，要实现上文中提到的抽象化效果，既离不开硬件的支持也离不开操作系统的支持（Memory Hierarchy）

更进一步，虚拟化能被推广为就是虚拟机（virtual machine），其本质是将 OS 当成应用程序在另一个 OS 上运行，这种运行在 OS 上的 OS 被称为客户操作系统（guest operating system）

虚拟机有两个巨大的优势：

  • 可以将仅能在其他 OS 上运行的应用程序搬到此 OS 上运行；

  • 可以使用虚拟机调试 OS；

Referee

It balances needs, separates conflicts, and facilitates sharing.

所谓 “裁判” 指的就是：保证所有活动相互独立的同时，合理分配硬件资源，即：

  • 资源分配（Resource allocation）— OS 需要合理的将硬件资源分配给各应用程序：

    • 不同的应用程序不仅能共享同一台计算机上的硬件资源，也能实现 “同时” 运行；

    • 没有程序能独占硬件资源，OS 需要确保各应用程序都有一定的硬件使用时间Referee-1；

  • 独立（Isolation）— OS 需要确保发生在应用程序 A 中的错误不会影响到同一计算机上的另一应用程序；

  • 交流（Communication）— OS 需要确保不同应用程序之间或同一计算机的不同用户之间可以交流；

Process

仅使用虚拟化将硬件资源加以抽象化显然是不够，还需要一个组织者将它们统合起来，这便是进程（Process）的概念，具体来说：

  • 应用会将提供自 OS 的进程当成是裸机，将其当成是运行环境；

  • 每个程序都运行在自己的进程中，顺便实现隔离；

  • 进程相比于裸机能提供各方面更优的接口；

Context Switch

每个进程在内存中都有着自己的进程描述符（process descriptor），操作系统以此为基础，确保在整体上呈现出多处理器

上下文切换时处理器中的值（寄存器）将被更新为另一进程的值

应用程序被读取到内存中之后才会变成进程，这其中包括进程结构分配操作（process structure allocate），进程也需要被放进调度队列（scheduler queue）中。等这一切都完成了，进程才是程序的实例化对象（instantiation）；基于此，实际上同一个应用程序可以对应多个进程，这些进程各自代表处于不同状态的应用程序，它们所做的工作自然也各不相同

某个进程 “拥有” 多少百分比的 CPU 指的是在一定时间之内，这个线程总共占用了 CPU 多少（百分比）的时间。不过，有时也会出现进程占用超过百分百的情况，这大概是由于在多核 CPU 上使用了单核 CPU 统计口径的缘故

Protection

当某一个进程试图访问内存中不属于它的部分或访问其他用户的数据（位于 Storage 中），OS 将会以（通常是）segmentation fault 的形式中断此进程

Glue

“胶水” 描述的是 OS 会提供标准化的共享服务（设定 IO 设备的接口、规定程序间通信的方式），以便规范化和简化应用程序的设计；让 OS 敲定应用程序间通信的方式可以减轻应用程序的负担（无需关注如何与其他应用进行通信），因此使相关的设计变得更加容易

操作系统所提供的通用服务不仅针对程序与程序，同时也能针对程序与硬件之间提供通用服务，其优势主要有两点：

  1. 应用程序无需关注硬件之间的细微差别，反之亦然；

  2. 应用程序和硬件的工作独立进行，两者并不相互绑定；
