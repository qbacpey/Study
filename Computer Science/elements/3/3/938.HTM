Title: Introduction : 定义 : Referee

It balances needs, separates conflicts, and facilitates sharing.

所谓 “裁判” 指的就是：保证所有活动相互独立的同时，合理分配硬件资源，即：

  • 资源分配（Resource allocation）— OS 需要合理的将硬件资源分配给各应用程序：

    • 不同的应用程序不仅能共享同一台计算机上的硬件资源，也能实现 “同时” 运行；

    • 没有程序能独占硬件资源，OS 需要确保各应用程序都有一定的硬件使用时间Referee-1；

  • 独立（Isolation）— OS 需要确保发生在应用程序 A 中的错误不会影响到同一计算机上的另一应用程序；

  • 交流（Communication）— OS 需要确保不同应用程序之间或同一计算机的不同用户之间可以交流；

Process

仅使用虚拟化将硬件资源加以抽象化显然是不够，还需要一个组织者将它们统合起来，这便是进程（Process）的概念，具体来说：

  • 应用会将提供自 OS 的进程当成是裸机，将其当成是运行环境；

  • 每个程序都运行在自己的进程中，顺便实现隔离；

  • 进程相比于裸机能提供各方面更优的接口；

Context Switch

每个进程在内存中都有着自己的进程描述符（process descriptor），操作系统以此为基础，确保在整体上呈现出多处理器

上下文切换时处理器中的值（寄存器）将被更新为另一进程的值

应用程序被读取到内存中之后才会变成进程，这其中包括进程结构分配操作（process structure allocate），进程也需要被放进调度队列（scheduler queue）中。等这一切都完成了，进程才是程序的实例化对象（instantiation）；基于此，实际上同一个应用程序可以对应多个进程，这些进程各自代表处于不同状态的应用程序，它们所做的工作自然也各不相同

某个进程 “拥有” 多少百分比的 CPU 指的是在一定时间之内，这个线程总共占用了 CPU 多少（百分比）的时间。不过，有时也会出现进程占用超过百分百的情况，这大概是由于在多核 CPU 上使用了单核 CPU 统计口径的缘故

Protection

当某一个进程试图访问内存中不属于它的部分或访问其他用户的数据（位于 Storage 中），OS 将会以（通常是）segmentation fault 的形式中断此进程
