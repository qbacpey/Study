UNIX has a system call, naturally enough called wait, that pauses the parent until the child finishes, crashes, or is terminated.

wait 会一直暂停父进程，直到子进程执行完成、崩溃或者终结

 

Since the parent could have created many child processes, wait is parameterized with the process ID of the child.

wait 的参数是子进程的 ID

With wait, a shell can create a new process to perform some step of its instructions, and then pause for that step to complete before proceeding to the next step. It would be hard to build a usable shell without wait
