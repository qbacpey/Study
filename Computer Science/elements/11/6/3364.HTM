各层的作用基本可归结为：

  • 应用层：将用户产生的数据打包成消息并发送到传输层；

    • 针对一类具体应用提供的协议；
    • FTP，SMTP，HTTP；

  • 传输层：使用 TCP/UCP 协议控制数据传输的速度等各种各样的参数，从而提高网络的可依赖性；

    • 主机到主机，端口到端口；
    • TCP，UDP；

  • 网络层：使用 IP 协议为数据添加一些元信息，并使用路由协议规划路由；

    • 源节点到目标节点；
    • IP，路由协议；

  • 链路层：根据数据包对应链路的类型，确定使用何种协议传输数据包；

    • 相邻节点之间的通信；
    • PPP，Ethernet；

  • 物理层：根据链路的物理媒介，选择相应的传输协议；

本质上，网络发送数据的过程实际上是一个不断将数据封装并加以传输的过程18：

  • 应用层将消息传递给传输层，即所谓应用层消息（application-layer message）；
  • 传输层对消息进行封装并在其上添加额外的信息19，生成传输层段（transport-layer segment）；
  • 网络层会将段加以封装，同时也在其中添加额外的数据20，生成网络层数据报（network-layer datagram）；
  • 链路层也会将额外的数据添加进去并生成链接层帧（link-layer frame）；
  • 总的来说，每层包都由以下两个部分组成：header field 和 payload field（后者就是从上层接受到的数据包）
  • 应用层

应用层（application layer）是网络应用及相关的协议所处的位置，网络应用会借助相关的协议与位于其它终端系统上的网络应用交换数据包

应用层中包含非常多的协议：

  • HTTP：提供网页文档的请求和传送；
  • SMTP：提供电子邮件的传送；
  • FTP：提供两个终端系统的文件传送；

应用层中的数据包被称为消息（message）

传输层

传输层（transport layer）提供不同终端系统间消息传递的服务

传输层中典型的传输协议有两种，TCP 和 UDP：

  • TCP 提供的服务是面向连接型（connection-oriented）的服务，功能主要包括以下四个：

    • 提供可靠，有序的字节流传输；
    • 流控制（flow control）21；
    • 将相对较长的消息分割成相对较短的段（segment）；
    • 提供拥塞控制（congestion-control）结构22；

  • UDP: UDP 提供的服务是无连接型的，相对于 TCP，其功能简单很多。既没有流控制，也没有拥塞控制，因此就不保证任何可依赖性；

如果传输层使用的是 TCP 协议，那么其中的数据包被称为报文段（segment）；如果使用的是 UDP 就称为数据报（datagram）

网络层

网络层（network layer）的功能是将一个主机上的数据报（datagram）传输到另一台主机上（从源节点到目标节点），本层的协议除了包括用于确定数据包路由（自起点到终点的路由）的路由协议，还包括 IP 协议

本层中的 IP 协议定义了数据报中的字段，以及终端系统以及路由器应当怎样处理这些字段

网络层中的数据包被称为数据报（datagram）

链路层

如果网络层想要将数据报转发到路由上的下一个节点，就必须借助链路层（link layer）的帮助：每个网络组件中的网络层都会将数据报传输给其下的链路层，再由它将数据报传送给下一个节点中的链路层。数据报到达后，节点链路层则会将接受到的数据报传送给其上的网络层

但是链路层具体会提供什么服务很大程度上取决于两节点之间所使用的链路形式及对应的协议。对于某一个数据报来说，其可能在一条链路上被以太网对应的链路层协议所处理，在另一条链路上被 PPP 对应的链路层协议所处理。因此，位于不同路由器上的网络层往往会收到不同链路层协议的服务

链路层中的数据包被称为帧（frame）

物理层

物理层（physical layer）传输对象的单位是帧中的单一一个 bit。类似于链路层，本层的协议（及服务）也依赖于链路的具体类型，更进一步的，甚至会依赖于链路上具体的传输介质（双绞线，光纤都有各自对应的协议）；所使用的协议不同，链路上 bit 的传输方式自然也不同

物理层的数据报就简单的被称为包（packet）
