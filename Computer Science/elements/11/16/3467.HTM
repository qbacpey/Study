Title: 链路层 : 交换机网络 : 链路层地址

链路层地址的作用是同一个 LAN 中不同的主机 NIC 或者路由器接口（下简称 “ NIC ”）相互通信的依仗。由于 NIC 和交换机没有通信需求，交换机没有链路层地址。涉及到与 LAN 之外的设备通信，就需要使用 IP 地址了。本小节将介绍以下内容：

  1. 链路层地址的语法和功能；
  2. 为什么同时拥有 IP 地址和链路层地址不可避免；
  3. ARP 如何将 IP 地址转化为链路层地址；

MAC 地址

链接层地址可被称为 LAN 地址（LAN address）、物理地址（physical address）或 MAC 地址（MAC address），对于大多数的局域网（LAN）来说，MAC 地址的长度都为 6 字节（48-bits）11

网卡与 MAC

主机和路由器接口本身实际上并没有 MAC 地址，真正拥有 MAC 地址的是它们的 NIC（网络接口）12；此外，交换机中直接和 NIC 相连的接口不含有 MAC 地址，这是因为交换机的职责是将数据报在 NIC 之间来回传输，而它会以一种透明地（transparently）方式完成这一任务13：

如上所见交换机实际上并不存在 MAC 地址，而只有 NIC 拥有 MAC 地址

扁平地址

前已述及，任何设备的 MAC 地址都是和硬件绑定并且不可更改、全球唯一的，此外 MAC 地址是扁平的（flat），这也就意味着 MAC 地址不像 IP 地址一样拥有层级结构；因此无论一块 NIC 去到哪里，它的 MAC 地址都不会发生改变

定向收发

当 NIC 想要向网络中另一块 NIC 发送数据帧的时候，发送方会将接收方的 MAC 地址插入到该帧之后再将其投放到 LAN 中。接收方在接受到网络中的某个数据帧之后，会检测该帧的目的 MAC 地址是否和自己的 MAC 地址相等，如果相等会提取出封装在其中的数据报并将其呈递给网络层；否则将直接丢弃此数据报14

广播收发

不过，有时候发送方也会希望将某个数据帧发送给 LAN 中的所有 NIC，这时发送方就会在数据帧的目的 MAC 地址处嵌入一段特殊的地址，这段地址被称为 MAC 广播地址（MAC broadcast address）15

ARP

地址解析协议（Address Resolution Protocol，ARP）的作用实际上就是将位于同一个 LAN中的 IP 地址，使用 ARP 表（ARP Table）将其转化为 MAC 地址16（虽然很多时候提到 ARP 都是指使用 ARP 包询问网络中的 MAC 地址，但是千万要记住。ARP 最主要的目的是将 IP 地址转换成 MAC 地址）

存在意义

为什么 NIC 不直接使用网络层的地址，而是自己单独使用一个 MAC 地址呢：

  1. LAN 需要支持任何网络层协议：网络层协议不仅包括 IP 协议，还有诸如 IPX、DECnet 等。换句话来说，如果 NIC 直接使用网络层地址（IP 地址）而不是 MAC 地址的话，它可能会很难支持其他的网络层协议；
  2. NIC 的地址无需频繁异动：考虑到 IP 地址经常随着位置的改变而发生改变，因此如果将 IP 地址当做链路层地址的话，很可能需要在 IP 地址发生改变的同时重新配置 NIC 的用于保存链路层地址的 RAM；
  3. 无需频繁将数据报呈递给网络层：如果撤销链路层地址这一设置，选择将所有接受到的数据帧都呈递给网络层的话，考虑到 LAN 中经常出现接收方并非本交换机的数据帧，那么网络层（也就是 CPU）需要频繁地被中断并检查该数据报（是否是发送给自己的数据报），这无疑将增加巨量的开销；

总而言之，链路层使用 MAC 地址而不是网络层 IP 地址的很大一部分原因是链路层需要实现与网络层的解耦合

维护方式

ARP 表实际上并不一定包含 LAN 中所有 IP-MAC 条目，此外这些条目都有一定的有效时间（一般为 20 分钟）

实践中，每一个 NIC 都拥有一个属于自己的 ARP 表，如果某个 NIC 想要将数据帧发送到网络上的另一个 NIC ，但是仅知道它的 IP 地址而不知道 MAC 地址，就会使用以下方式获取 MAC 地址：

  1. 发送方会生成一个 ARP 包，其中包含：发送方和接收方的 IP 地址和 MAC 地址17；
  2. 发送方会将此包的目的 MAC 地址设置为 MAC 广播地址（FF-FF-FF-FF-FF-FF），以便网络上所有的 NIC 都能接受到这个 ARP 包（网络上所有的 NIC 都会接受到此 ARP 包）；
  3. 接受到此 ARP 包的 NIC 会将此 APR 包传递给其 ARP 模块（查询本机 APR 表），检查本机 IP 地址是否为 ARP 包中寻求的 IP 地址。如果是，就打一个 ARP 包返回给发送方（利用该 ARP 包中发送方的 MAC 地址作为目的 MAC 地址）；

因此，尽管发送方发送的是一个广播包，但是实际上只有一个 NIC 会响应该 ARP 包（广播帧-标准帧）

此外，上述便是 ARP 表的自我建立功能的体现

ARP 与网络协议栈

APR 究竟是哪一层的协议：

  1. 考虑到 ARP 包会被封装到链路层帧中，它似乎属于链路层的上层；
  2. 考虑到 ARP 包中含有 MAC 地址，因此它似乎是链路层协议；
  3. 考虑到 ARP 包中含有 IP 地址，它似乎是网络层协议；

综合来看，ARP 似乎应该被当成一个位于网络层和链路层中间的协议

跨 LAN 传输

如果某个 LAN 中的设备想要将数据帧发送到另一个 LAN 中，会发生下面的事情（注意，每个路由器接口都有一个 NIC 和 ARP 模块）：

  1. 发送方会发送一个数据帧，其 MAC 地址是路由器的 MAC 地址，IP 地址是接受方的 IP 地址；
  2. 路由器的会将上述数据帧解包，根据其中包含的 IP 地址，结合自己的路由表，推断出接下来要将此数据报发送到哪个接口中。同时，如果路由器发现接下来转发的位置并不是另一个路由器，而是一个 LAN，那么就会根据目标 IP 地址，查询 ARP 表，确定目标主机的 MAC 地址，并重新封装这个数据报；
