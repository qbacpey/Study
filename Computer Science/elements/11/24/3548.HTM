EXAMPLE: Is a kernel interrupt handler a thread?

ANSWER: No, an interrupt handler is not a thread. A kernel interrupt handler shares some resemblance to a thread: it is a single sequence of instructions that executes from beginning to end.

内核中断处理器并非线程，尽管它确实与线程共享某些特性（从头执行到尾的指令序列）

However, an interrupt handler is not independently schedulable: it is triggered by a hardware I/O event, rather than a decision by the thread scheduler in the kernel. Once started, the interrupt handler runs to completion, unless preempted by another (higher priority) interrupt. □

内核中断处理器并非线程，本质原因是中断处理器并非独立可调度，它为硬件 IO 事件驱动而非内核中的线程调度器所驱动。一旦启动，中断处理器就会一直运行直到执行完毕，除非被其他更高优先级的中断抢先执行

线程独立可调度的意思是可以随时被调度器打断，将 CPU 让给其他线程

言下之意就是，线程必须独立可调度，也就是能被线程调度器随时打断其执行过程

为什么中断处理程序并非独立可调度？因为中断处理程序为中断而非调度器所驱动
