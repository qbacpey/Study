Title: 链路层 : 交换机网络

基于交换机的 LAN（Switched LAN）能在很大程度上解决广播频道中存在的问题，最为突出的体现是 MAC 协议本身变得没有什么意义了。本小节将会涉及以下四个内容：

  • 链路层地址；
  • 以太网协议；
  • 链路层交换机；
  • 大规模交换机 LAN；

链路层地址

链路层地址的作用是同一个 LAN 中不同的主机 NIC 或者路由器接口（下简称 “ NIC ”）相互通信的依仗。由于 NIC 和交换机没有通信需求，交换机没有链路层地址。涉及到与 LAN 之外的设备通信，就需要使用 IP 地址了。本小节将介绍以下内容：

  1. 链路层地址的语法和功能；
  2. 为什么同时拥有 IP 地址和链路层地址不可避免；
  3. ARP 如何将 IP 地址转化为链路层地址；

MAC 地址

链接层地址可被称为 LAN 地址（LAN address）、物理地址（physical address）或 MAC 地址（MAC address），对于大多数的局域网（LAN）来说，MAC 地址的长度都为 6 字节（48-bits）11

网卡与 MAC

主机和路由器接口本身实际上并没有 MAC 地址，真正拥有 MAC 地址的是它们的 NIC（网络接口）12；此外，交换机中直接和 NIC 相连的接口不含有 MAC 地址，这是因为交换机的职责是将数据报在 NIC 之间来回传输，而它会以一种透明地（transparently）方式完成这一任务13：

如上所见交换机实际上并不存在 MAC 地址，而只有 NIC 拥有 MAC 地址

扁平地址

前已述及，任何设备的 MAC 地址都是和硬件绑定并且不可更改、全球唯一的，此外 MAC 地址是扁平的（flat），这也就意味着 MAC 地址不像 IP 地址一样拥有层级结构；因此无论一块 NIC 去到哪里，它的 MAC 地址都不会发生改变

定向收发

当 NIC 想要向网络中另一块 NIC 发送数据帧的时候，发送方会将接收方的 MAC 地址插入到该帧之后再将其投放到 LAN 中。接收方在接受到网络中的某个数据帧之后，会检测该帧的目的 MAC 地址是否和自己的 MAC 地址相等，如果相等会提取出封装在其中的数据报并将其呈递给网络层；否则将直接丢弃此数据报14

广播收发

不过，有时候发送方也会希望将某个数据帧发送给 LAN 中的所有 NIC，这时发送方就会在数据帧的目的 MAC 地址处嵌入一段特殊的地址，这段地址被称为 MAC 广播地址（MAC broadcast address）15

ARP

地址解析协议（Address Resolution Protocol，ARP）的作用实际上就是将位于同一个 LAN中的 IP 地址，使用 ARP 表（ARP Table）将其转化为 MAC 地址16（虽然很多时候提到 ARP 都是指使用 ARP 包询问网络中的 MAC 地址，但是千万要记住。ARP 最主要的目的是将 IP 地址转换成 MAC 地址）

存在意义

为什么 NIC 不直接使用网络层的地址，而是自己单独使用一个 MAC 地址呢：

  1. LAN 需要支持任何网络层协议：网络层协议不仅包括 IP 协议，还有诸如 IPX、DECnet 等。换句话来说，如果 NIC 直接使用网络层地址（IP 地址）而不是 MAC 地址的话，它可能会很难支持其他的网络层协议；
  2. NIC 的地址无需频繁异动：考虑到 IP 地址经常随着位置的改变而发生改变，因此如果将 IP 地址当做链路层地址的话，很可能需要在 IP 地址发生改变的同时重新配置 NIC 的用于保存链路层地址的 RAM；
  3. 无需频繁将数据报呈递给网络层：如果撤销链路层地址这一设置，选择将所有接受到的数据帧都呈递给网络层的话，考虑到 LAN 中经常出现接收方并非本交换机的数据帧，那么网络层（也就是 CPU）需要频繁地被中断并检查该数据报（是否是发送给自己的数据报），这无疑将增加巨量的开销；

总而言之，链路层使用 MAC 地址而不是网络层 IP 地址的很大一部分原因是链路层需要实现与网络层的解耦合

维护方式

ARP 表实际上并不一定包含 LAN 中所有 IP-MAC 条目，此外这些条目都有一定的有效时间（一般为 20 分钟）

实践中，每一个 NIC 都拥有一个属于自己的 ARP 表，如果某个 NIC 想要将数据帧发送到网络上的另一个 NIC ，但是仅知道它的 IP 地址而不知道 MAC 地址，就会使用以下方式获取 MAC 地址：

  1. 发送方会生成一个 ARP 包，其中包含：发送方和接收方的 IP 地址和 MAC 地址17；
  2. 发送方会将此包的目的 MAC 地址设置为 MAC 广播地址（FF-FF-FF-FF-FF-FF），以便网络上所有的 NIC 都能接受到这个 ARP 包（网络上所有的 NIC 都会接受到此 ARP 包）；
  3. 接受到此 ARP 包的 NIC 会将此 APR 包传递给其 ARP 模块（查询本机 APR 表），检查本机 IP 地址是否为 ARP 包中寻求的 IP 地址。如果是，就打一个 ARP 包返回给发送方（利用该 ARP 包中发送方的 MAC 地址作为目的 MAC 地址）；

因此，尽管发送方发送的是一个广播包，但是实际上只有一个 NIC 会响应该 ARP 包（广播帧-标准帧）

此外，上述便是 ARP 表的自我建立功能的体现

ARP 与网络协议栈

APR 究竟是哪一层的协议：

  1. 考虑到 ARP 包会被封装到链路层帧中，它似乎属于链路层的上层；
  2. 考虑到 ARP 包中含有 MAC 地址，因此它似乎是链路层协议；
  3. 考虑到 ARP 包中含有 IP 地址，它似乎是网络层协议；

综合来看，ARP 似乎应该被当成一个位于网络层和链路层中间的协议

跨 LAN 传输

如果某个 LAN 中的设备想要将数据帧发送到另一个 LAN 中，会发生下面的事情（注意，每个路由器接口都有一个 NIC 和 ARP 模块）：

  1. 发送方会发送一个数据帧，其 MAC 地址是路由器的 MAC 地址，IP 地址是接受方的 IP 地址；
  2. 路由器的会将上述数据帧解包，根据其中包含的 IP 地址，结合自己的路由表，推断出接下来要将此数据报发送到哪个接口中。同时，如果路由器发现接下来转发的位置并不是另一个路由器，而是一个 LAN，那么就会根据目标 IP 地址，查询 ARP 表，确定目标主机的 MAC 地址，并重新封装这个数据报；

以太网

以太网（Ethernet）作为一种被广泛应用同时也历史悠久的 LAN 技术，拥有一条清晰的发展脉络：

  1. 原始以太网：广播 LAN（Broadcast LAN）指的是 LAN 中任何数据帧都会到达 LAN 上的所有连接到总线上的 NIC 并被其处理，基于总线拓扑和基于集线器星型拓扑实现的以太网都是广播 LAN 的一种18；
  2. 基于集线器的以太网：集线器（hub）是一个针对单个 bit 进行操作的物理层设备，其功能仅仅是强化任何接受到的 bit 信号（无论是 0 还是 1）并将此 bit 广播到其他的所有接口上。特别的。如果一个集线器同时接收到来自两个接口的信号，冲突就会发生。这种类型的以太网依然属于广播 LAN；
  3. 基于交换机的以太网：交换机（switch）作为一种如同路由器一般的储存-转发设备，从根源上解决了网络冲突。同时，这种以太网也弃用了广播频道，转而大量选择 PPP 网络；

以太网帧结构

以太网帧结构（Ethernet Frame Structure）基本上在以太网被提出以来就没有发生改变，历史上任何使用以太网的结构都使用了同样格式的以太网帧（虽然 VLAN 中加了点东西）19，它由六个部分组成：

  • 数据字段（Data field）是以太网帧中实际承载数据的区域，它具有以下长度限制：

    • 最大值不得超过 1500 字节：如果超过 1500 字节，主机必须对其加以分段；
    • 最小值不得小于 46 字节：如果小于 46 字节，数据字段必须被填充到 46 字节。接收方网络层可以通过位于（IP）数据报头字节中的长度字段（它们标识了数据报的原始长度），剔除这部分多余的填充字段；

  • 目的地址（Destination address）中保存着目标 NIC 的 MAC 地址，考虑到这是以太网，其长度为 6 字节。针对不同情况，接收方有两种应对方案：

    • 如果接收方发现该包的目的地址是本机的 MAC 地址，或者发现是广播 MAC 地址，就会将该帧中的数据字段发送到网络层；
    • 任何其他的情况，接收方都会直接将此帧丢弃；

  • 源地址（source address）保存着发送方的 MAC 地址，因此长度也是 6 字节；

  • 类型字段（type filed）通过标识发送方网络层所使用的协议，允许接收方以太网多工复用（multiplex）不同的网络层协议，长度为 2 字节。

    • 所谓多工复用，实际上指的就是同一个主机上有多种不同的网络层协议（不仅仅包括 IP 协议）因此以太网需要正确地与不同的网络层协议进行对接。例如说当以太网帧到达接收方时，相应的 NIC 需要知道接下来应该将其中的数据字段呈递（解多工复用，demultiplex）给哪个网络层协议20；

  • 循环冗余校验（CRC）长度为 4 字节，允许接收方 NIC 检测帧中是否出现错误；

  • 前同步信号（preamble）是一个 8 字节长的字段，前 7 个字节是 10101010（前同步码），最后一个字节是 10101011（帧开始定界符） 简单来说，该字段有两个作用：

    • 时钟同步：接收方能根据前 7 个字节的传输速率，确定发送方的数据传输速度21；
    • 通知：前同步信号的最后两个比特（11）的用处就是通知接收方 “重要的数据要来了”；

总的来说，以太网帧的数据字段为 46~1500 字节，其外包装必然为 18 字节22。因此有效 MAC 帧长度为 64~1500 字节

服务类型

以太网所提供的服务具有两个特点：

  • 无连接（connectionless）的服务，也就是说，两个实现以太网协议 NIC 在传输数据之前不会握手；
  • 不可靠（unreliable）：当接收方对数据帧执行 CRC 校验时，既不会在成功时通知发送方，也不会在失败时通知发送方。不可靠传输一方面使得以太网简单也易于实现，另一方面会使以太网传输数据流出现断点23；
  • 无流量控制：以太网并不提供流量控制的服务，因此如果网络层从 NIC 中读取数据的速度不够快，NIC 的缓存很可能会直接溢出；
  • 错误检测：尽管以太网确实执行错误检测（在使用 CRC 检测到错误时丢弃该数据帧）但是以太网并不执行错误修正；
  • 全双工与半双工：如果以太网使用链路层交换机进行维护，那么以太网就能提供全双工的服务（因为交换机既支持全双工也支持半双工）；

那么，接收方的应用能否察觉这些断点？实际上，这取决于应用是使用 UDP 协议还是 TCP 协议。前者代表应用最终会看到数据出现了断点；后者则不会，这实际上意味着发送方将会再次发送这个被链路层丢弃的包24

广播 LAN

前面提到，原始以太网和基于集线器实现的以太网都是广播型 LAN，具体到实践中，这种以太网具有如下特点：

  1. 由于以太网帧长度不一，因此使用的是无时隙的 CSMA/CD 协议（半双工）实现；
  2. 遵循 10BaseT 协议，使用双绞铜线，传输速率为 10Mbps；
  3. 帧间最小间隔为 ，相当于 96-bit 的发送时间：节点在检测到总线空闲之后，还需要等这么一小段时间，让接收到数据帧的节点清理接收缓存并做好接收下一帧的准备25；
  4. 使用集线器组织网络：由于集线器本身的限制，网络中任意两台终端设备的最大距离只能是 1km；
  5. 不能进行全双工通信，只能进行双向交替通信（半双工）；
  6. 使用二进制指数退避算法计算退避时间；

争用期

以太网还有一个争用期的概念，就是这个东西决定了以太网帧的最小长度为什么是 64 字节：

  1. 极端情况下（冲突发生位置和节点间距离为 1km）发送数据的节点如果想要检测到冲突，其一是需要等到信号传播到冲突发生的位置（1km），其二是需要等到冲突信号回到节点（1km）上述两项时间加起来就是一个节点检测到冲突发生所需要的最大时间（），这便是以太网的争用期26。如果在这段时间内节点没有检测到冲突的发生，就代表整个网络中没有冲突发生；
  2. 那么，我们要怎样做才能确保不会出现 “数据传输到网络的某个部分的时候出现了冲突，但是节点以为自己在传输数据的过程中没有出现冲突” 这一现象呢？实践中，以太网将帧的最小长度定义为了 “传输最小长度的数据帧所需要的时间，就是以太网争用期的时间”，这就是帧最小长度为 64 字节的来源。如果节点在传输这 64 个字节的过程中网络中没有出现冲突，那就代表传输数据的过程中确实不存在冲突，节点可以放心大胆的将后续的所有数据一并发送到网络中27；

物理层

以太网的协议既对链路层做出了定义，也对物理层做出了定义。涉及其中链路层部分的标准（MAC 协议、数据帧格式）始终未变，而涉及到物理层的标准却随着介质的变化一直在发生改变：

以太网在使用总线拓扑和基于星型拓扑的时代，毫无疑问地是冲突频发、需要 MAC 协议调配网络的广播链路。但是现如今，新式的以太网大多通过基于交换机的星型拓扑实现。这种存储-转发式交换机会协调传输，并确保不会同时将两个数据帧转发到同一个接口上（同时它也是全双工的，这也意味着双向传输不会相互干扰）

这实际上也就意味着，在新式的以太局域网中并不会出现冲突，自然也没有使用 MAC 协议的必要了

Gbit 以太网

这种类型的以太网使用标准以太网帧格式，同时支持点对点链路和共享广播频道：

  • 共享广播频道：

    • CSMA/CD 类型链路；
    • 需要节点之间的距离相对较短；

  • 共享广播频道：

    • 全双工；
    • 40 Gbps；

交换机

交换机（Switch）最本质的作用是将接受到的数据帧，以一种不会引发数据冲突的方式分发到其他 NIC 上，为此将涉及到方方面面的问题：

  1. 交换机所服务的范围有多大？
  2. 交换机怎样确定将数据帧转发的方向？
  3. 交换机如何确保网络中不会出现冲突？

总的来看，交换机具有如下特点：

  1. 全双工；
  2. 自学习；
  3. 即插即用；
  4. 透明；

转发和过滤

过滤（filtering）指的是交换机会确定将数据帧推送到某个接口上还是直接将该数据帧丢弃

转发（forwarding）指的是交换机需要确定应当将数据帧发送到哪一个接口上，并将数据帧切实的传递到该接口上

无论是过滤还是转发，都要依靠交换表（switch table）来实现这一功能。其中的条目一般具有以下三个性质：

  1. MAC 地址；
  2. 与 MAC 地址对应的接口标识；
  3. 条目被添加到交换表中的时间（似乎不是条目的有效时间）；

转发方式

值得注意的是，如果交换机要将某个数据帧转发到某个接口对应的 LAN 段的话，需要将该数据帧放到该接口对应的缓存中

假设数据帧从交换机的接口 x 中被传递过来，目标 MAC 地址是 DD-DD-DD-DD-DD-DD，那么过滤和转发工作基本可以归结为以下情形：

  1. 交换表无目标 MAC 地址：交换机会将这个数据帧广播到所有（除了来源交换机接口 x）交换机接口中；
  2. 对应于目标 MAC 地址的交换机接口就是 x28：交换机会直接丢弃掉这个数据包；
  3. 对应于目标 MAC 地址的交换机接口不是 x：交换机需要将该数据帧推送到与 y 相关联的 LAN 段中，它会将数据帧放到位于 y 对应的缓存中；

另外，如果交换机需要向 LAN 段中传递数据的话，会使用 CSMA/CD 协议

自学习

交换机的自学习（self-leaning）能力基本可以归结如下：

  1. 每当一个数据帧到达交换机时，它会将以下信息保存到交换表中：

    1. 数据帧来源主机的 MAC 地址29；
    2. 数据帧到达的接口；
    3. 数据帧到达的时间；

  2. 如果一段时间没有新的数据帧从某一个 MAC 地址中到达，交换机表中涉及到该地址的条目会被移除30；

此外，交换机同时也是即插即用型设备（plug-and-play），如果想要在网络中添加交换机，唯一需要做的就是将交换机连接到 LAN 段中。交换机也是全双工（full-duplex）的设备，任何交换接口都能同时收发信息

优势

交换机相比于广播链路，具有如下优势：

  1. 无需担心冲突问题：交换机会缓存数据帧并确保不会同时向一个 LAN 段中传输两个数据帧；
  2. 多态链路：由于交换机将不同 LAN 段之间予以隔离，因此使用交换机就能让不同类型的网络进行协作；
  3. 管理：交换机能让网络管理员的工作更加轻松

VLAN

交换机层级结构指的是先用一些交换机将终端设备连接起来，构成一个交换机式的 LAN。随后再使用另外的交换机将这些 LAN 和其他服务器连接起来，这种网络结构的问题（或者说使用 VLAN的优点）包括以下两点：

  1. 缺少流量独立性：考虑到某个局域网中的广播流量（或者说本应广播给该 LAN 中所有设备的数据帧）会经由上层的交换机传递给其他 LAN，这不仅会对整个网络的性能有损耗，更重要的是数据安全性不能得到保证；
  2. 用户管理问题：如果需要将用户从一个 LAN 迁移到另一个 LAN 中，必须重新插拔双绞线；

以上问题只需要使用 VLAN 就能立刻得到解决

VLAN 的工作方式

支持 VLAN（virtual local area network）的交换机允许用户对其端口加以划分，使这些端口（及与之相连的设备）分属不同的 VLAN：

VLAN 的使用在很大一部分程度上解决了 LAN 中原本存在的问题：

  1. VLAN 本身就形成了一个具有严格限制的广播域（broad cast），任何其他广播域中的广播流量都不能传递到本广播域中，本广播域中的任何广播流量也不能传递到其他广播域中；
  2. 如果网络管理员想要将某个用户更改到另一个 VLAN 中，无需移动双绞线，仅仅只需修改 VLAN 的配置即可；

不过，虚拟局域网仅仅只是局域网提供给用户的一种服务，并不是一种新型的局域网

跨 VLAN 通信

那么，不同 VLAN 之间要如何实现通信呢？

让分属不同的 VLAN 的端口外接路由器。此时，从 VLAN A 到 VALN B 的数据报必须经由此端口传递到路由器中，再由路由器将该数据报转发回 VLAN B

不过很快，交换机厂商就将路由器和交换机集成起来，因此也就不需要外接路由器了

小结

链路层中含有两种主流的网络：

  1. 广播网络；
  2. 交换机网络；

一些背后的东西：

  1. 集线器所分割的是不同的网段（network segment），交换机所分割的对象是不同的冲突域（collision domain），路由器分割的是不同的 LAN。很显然，一个 LAN 中可以拥有多个交换机、多个冲突域；

  2. 以太网 LAN 内设备传输数据，需要借助 MAC 地址确定投递数据帧的方向。更大区域的通信则需要通过 IP 地址实现。涉及到这方面的行为可归纳如下：

    1. NIC 想要将数据发送到 LAN 中的其他 NIC ：如果其 ARP 表中并没有目的地相关条目，则会发送一个广播 ARP 包询问网络上的 NIC ，获得目的 MAC 地址之后再发送地址定向的数据帧；

  3. 交换机相关行为：

    1. 对于任何接收到的数据帧，交换机都会读取数据帧来源 MAC 地址和对应的交换机接口并将这个条目保存到交换表中；
    2. 对于不知道目标 MAC 地址对应交换机接口的数据帧：交换机会直接将这个帧广播到其他所有的接口中。此外，目标地址是广播地址的数据帧也会被直接这样处理；
    3. 对于知道目标 MAC 地址对应的交换机接口的数据帧：交换机会直接将数据推送到对应接口中；
    4. 尽管交换机表中确实有相关条目，但是条目中指示的接口就是数据帧的来源接口：这时候实际上就代表该接口对接的是一个拥有多个主机的冲突域，并且这个冲突域采用的可能是广播链路而不是点对点链路。这时候交换机会直接将这个数据帧忽略掉；

整体逻辑：

  1. LAN 的划分并不是通过交换机或者集线器进行划分的，而是通过路由器进行划分的。这一点可以从两个方面看出来：

    1. 路由器会将传递给自己的数据帧解包，根据其中包含的 IP 地址，结合自己的路由表，推断出接下来要将此数据报发送到哪个接口中。同时，如果路由器发现接下来转发的位置并不是另一个路由器，而是一个 LAN，那么就会根据目标 IP 地址，查询 ARP 表，确定目标主机的 MAC 地址；
    2. 在 VLAN 的学习过程中，一种在不同 VLAN 之间进行通信的方式是外接路由器，让一个端口同时属于两个 VLAN。这样如果这两个 VLAN 之间想要进行通信的话，就必须经由这个端口将数据发送到路由器中再经由路由器将数据转发回来；

  2. 路由器接口或者终端设备使用 ARP 表维护目的地址，交换机使用交换表维护目的地址：

    1. ARP 表中仅会保存其所属 LAN 中其他 NIC 的 MAC 地址，如果想要将数据转发到外界 LAN ，需要将目标 MAC 地址设置为路由器接口的 MAC 地址，让路由器利用目标 IP 地址将数据转发到其他 LAN 中；
    2. 交换机表中会保存所属 LAN 中其他 NIC 的 MAC 地址和交换机接口的对应关系，但并不会保存 IP 地址，仅用于在 LAN 内部转发数据帧；

  3. 只有 NIC 拥有 MAC 地址，交换机没有 MAC 地址。本质上是因为没有任何向交换机发送数据帧的需求，在主路的眼中，自己所处的 LAN 中实际上并不存在交换机这个东西；
