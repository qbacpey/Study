Without the stub, a user program that returned from main would try to pop the return program counter, and since there is no such address on the stack, the processor would start executing random code

当 main 函数返回时，它会试图弹出并跳转到储存在用户栈中的返回地址。如果让 main 函数成为整个进程的入口，那么此时的栈中并不会存有 main 的返回地址，因此此时 main 所弹出的“返回地址”将会是一个垃圾值，导致进程跳转到随机内存位置

main 函数的返回不能视为进程的结束，显式调用 exit 才能视为进程的结束

不能让 main 作为进程入口的原因是什么？进程没有显式调用 exit 终结进程
