GDB 中的 automatic display list 会在什么时候被打印？

GDB 每一次停顿的时候

有两种 “程度” 的自动打印：

  • 每次程序执行被阻塞（断点，next 等等）时都会打印一些数据，核心是 automatic display list；

  • 程序被指定断点阻塞时，GDB 执行附着在该断点上的指令，核心是 条件断点；

前者可以理解为在程序每一次停顿的时候都将 auto-list 中处于启用状态的变量、按照设置时的格式、用类似于执行print的方式，打印出来。有三种设置 auto-list 的格式，含义各不相同：

如果我们想让 GDB 停顿时以指定格式打印某个表达式或者地址，可以执行什么指令？display/fmt expr

  • display expr：将expr添加到 auto-list 中，停顿时打印；

  • display/fmt expr：[fmt]和这里提供的参数相同，停顿时等价于执行 p/fmt expr；

  • display/fmt addr：[fmt]特指i,s，停顿时等价于执行x/fmt addr；

  • info display；

  • display；

另外，当 auto-list 中出现函数的本地变量时，在脱离该变量的作用域之后会自动禁用此变量，下一次再进入此变量作用域之后又能重新启用

执行什么指令手动打印 automatic display list？display

如何预览当前的 automatic display list？info display

执行什么语句启用 automatic display list 中的某个变量？enable display <dnums>

执行什么语句禁用 automatic display list 中的某个变量？disable display <dnums>
