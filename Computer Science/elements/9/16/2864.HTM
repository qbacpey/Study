After a new child process is created, both processes will execute the next instruction following the fork() system call.

当 fork 函数返回之后，所有进程都会？执行 fork 的下一条指令

Therefore, we have to distinguish the parent from the child. This can be done by testing the returned value of fork():

调用 fork 之后，区分当前进程以及判断是否成功创建子进程的方式是通过检查 fork 的返回值

    • If fork() returns a negative value, the creation of a child process was unsuccessful.

  • 如果 fork 返回负数，那么就代表子进程创建失败

    • fork() returns a zero to the newly created child process.

  • 如果 fork 返回 0，那么就代表当前进程是新创建的子进程
    • fork() returns a positive value, the process ID of the child process, to the parent.

  • 如果 fork 返回正数，该正数就是子进程的进程ID，同时代表当前进程是父进程

The returned process ID is of type pid_t defined in sys/types.h.

fork 所返回的进程 ID 类型是 pid_t，类型定义位于 sys/types.h 中

 

Normally, the process ID is an integer.

一般来说，进程 ID 都是一个整数

Moreover, a process can use function getpid() to retrieve the process ID assigned to this process

如果想要获取当前进程的 ID，可以执行 getpid() 函数
