Once the handler starts running, it can use the pushad (“push all double”) instruction to save the remaining registers onto the stack.

当中断处理程序运行时，它可以选择使用 `pushad` 指令将余下的所有寄存器保存到内核中断栈中

This instruction saves all 32-bit x86 integer registers. On a 16-bit x86, pusha is used instead. Because the kernel does not typically perform floating point operations, those do not need to be saved unless the kernel switches to a different process

之所以让中断处理程序选择是否要将剩余的寄存器（大多数是浮点数寄存器）保存到栈中，是因为内核的执行不需要浮点数寄存器，因此只有当中断处理程序切换执行其他用户进程时才需要保存浮点数寄存器
