Implementing threads : User-level thread scheduling :

Title: Implementing threads : User-level thread scheduling : Cooperative scheduling of user-level threads

The cooperative model is similar to multiprogramming where a process executes on the CPU until making a I/O request.

多道程序系统上的进程会一直执行直到发出 IO 请求让出 CPU 时间

Cooperative user-level threads execute on the assigned kernel-level thread until they voluntarily give back the kernel thread to the thread manager.

协作式调度中的用户级线程会一直在内核线程上执行，直到它自愿让出内核线程

In the cooperative model, threads yield to each other, either explicitly (e.g., by calling a yield() provided by the user-level thread library) or implicitly (e.g., requesting a lock held by another thread).

协作式调度中线程既可以通过显式的方式让出 CPU（yield）也可以隐式的让出 CPU（请求一个为其他线程持有的锁）

调度模式指的是抢先调度、协作调度、混合调度这三种；线程系统模式指的是一对一、多对一、多对多、两层这四种
