Title: 链路层 : 交换机网络 : 以太网 : 广播 LAN : 争用期

以太网的争用期决定了以太网帧的最小长度为什么是 64 字节：

  1. 极端情况下（冲突发生位置和节点间距离为 1km）发送数据的节点如果想要检测到冲突，其一是需要等到信号传播到冲突发生的位置（1km），其二是需要等到冲突信号回到节点（1km）上述两项时间加起来就是一个节点检测到冲突发生所需要的最大时间（2t），这便是以太网的争用期（10BaseT 类型以太网争用期长度为 51.2 us​，相当于 512-bit 发送所用时间）。如果在这段时间内节点没有检测到冲突的发生，就代表整个网络中没有冲突发生；
  2. 那么，我们要怎样做才能确保不会出现 “数据传输到网络的某个部分的时候出现了冲突，但是节点以为自己在传输数据的过程中没有出现冲突” 这一现象呢？实践中，以太网将帧的最小长度定义为了 “传输最小长度的数据帧所需要的时间，就是以太网争用期的时间”，这就是帧最小长度为 64 字节的来源。如果节点在传输这 64 个字节的过程中网络中没有出现冲突，那就代表传输数据的过程中确实不存在冲突，节点可以放心大胆的将后续的所有数据一并发送到网络中（如果节点在传输这 64 个字节的过程中没有检测到任何冲突，考虑到频道上已经被这个数据帧占满了，而以太网又是使用的载波监听，因此确实可以高枕无忧的将所有剩余数据传输到网络上；如果节点在传输数据的过程中检测到了冲突，必然是在传输前 64 个字节的过程中。考虑到帧的最小长度就是 64 字节，必然还有剩余数据等待发送，不会出现将数据帧完整传递到频道上之后才出现冲突（必然是在传输过程中出现冲突），因此无论如何都不会出现那种情况）；
