Transition from RUNNING to idle. When a user-level activation finds an empty ready list (i.e., it has no more work to do), it can make a system call into the kernel to return the virtual processor for use by some other process.

当用户级激活（调度器？）发现准备队列为空，它就会执行系统调用，并将虚拟处理器返回给内核

什么时候用户级线程调度器会将内核进程交还给内核？准备队列为空

用户级线程调度器如何将内核线程交还给内核？执行系统调用

当内核接收到用户级线程调度器交还的内核线程，它能否将这个内核线程交给新的用户进程？可以

调度器激活实现中，用户级线程系统有自己的调度队列、每个用户级线程在用户进程中也有自己的 TCB，这些东西和内核线程的 TCB 和调度队列是独立的
